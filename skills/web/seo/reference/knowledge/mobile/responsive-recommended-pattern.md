# Responsive design is Google's recommended mobile-first pattern

Google names responsive web design — a single URL serving one HTML source that adapts by screen size — as the recommended pattern because it is easiest to maintain and structurally avoids the content/link/metadata divergence that separate-URL (m-dot) or dynamic-serving setups risk.

## Facts
- Google states responsive web design is recommended "because it's the easiest design pattern to implement and maintain." [google-mobile]
- Responsive = one URL and one HTML source for all devices; this consolidates link equity to a single URL, avoids duplicate-content issues, and simplifies crawl architecture. [digitalapplied][clickrank]
- Separate mobile URLs (m-dot) and dynamic serving are still supported but push the parity burden onto the site owner (equivalent content, rel=canonical/alternate, matching metadata) — responsive removes that class of failure by construction. [google-mobile]
