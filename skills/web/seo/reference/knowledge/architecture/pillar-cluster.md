# Pillar-and-cluster (topical hub) internal-link structure

The pillar-and-cluster model — a hub pillar page interlinked with every supporting cluster page (and clusters linked to each other) — is the consensus 2026 internal-link topology for signalling topical authority; the structural pattern is sound but the vendor traffic/word-count magnitudes are unproven.

## Facts
- Structure (established direction): a pillar page covering a topic broadly links out to each cluster/detail page; every cluster page links back to the pillar and to relevant sibling clusters, forming a dense topical hub. [vendor]
- Rationale: Google increasingly evaluates topical competence across a whole section/domain rather than isolated pages, so comprehensive interlinked coverage helps deeper pages rank and concentrates link equity within the topic. [vendor]
- Do NOT tie this to a word count. Vendor claims of "pillars need 3,000–5,000 words" or "clusters give +40% traffic" are single-source magnitudes with no Google backing — word-count minimums are a MYTH (see foundations/2026-seo-myths). Recommend the link topology, never a word target. [vendor]
- This is a content/architecture RECOMMENDATION (flag-and-advise), not an auto-fix: building real pillar/cluster pages is editorial work, not a mechanical link insertion. [vendor]
