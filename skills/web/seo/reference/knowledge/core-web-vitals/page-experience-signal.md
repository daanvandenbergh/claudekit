# Page experience is a minor / tie-breaker ranking signal

Google treats page experience — of which Core Web Vitals are one part — as a light, no-single-signal ranking input that mostly breaks ties between comparably relevant pages; a highly relevant page with mediocre CWV can still outrank a fast but less relevant one.

## Facts
- Google: "There is no single signal" — core ranking systems look at many signals aligning with overall page experience; good CWV/Search Console results do NOT guarantee top ranking. [Google Search Central, page-experience]
- Google prioritizes relevance: "Google Search always seeks to show the most relevant content, even if the page experience is sub-par." Page experience matters most as a differentiator when lots of comparably helpful content competes. [Google Search Central, page-experience]
- Practitioner framing (secondary, John Mueller): CWV act as a tie-breaker, not a gate. Consistent with the primary doc's "modest weight / relevance first" language. [confidence: high for "minor signal"; the exact "tie-breaker" wording is a secondary characterization]
